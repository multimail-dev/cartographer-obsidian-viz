Information Model

What the system knows, how knowledge transforms, and how it feeds back into itself

Knowledge Lifecycle

Every piece of knowledge enters as a raw Obsidian note and is progressively transformed through five stages. Each stage feeds the next — and the later stages feed back into note creation, closing the loop.

1. Raw Note
Markdown file in R2 with YAML frontmatter.
Required: type tags[] created
Optional: speakers, concepts, aliases
→
2. Extraction
extractEdgesFromNote() parses:
• wikilinks from body text
• tags from frontmatter
• speakers concepts from frontmatter
• section context (## Related, ## Speakers, ## Key Claims → typed edges)
→
3. Graph
Nodes (vault_nodes) + Edges (vault_edges).
Degree computation (in/out).
FTS5 full-text index (vault_fts).
Audit log of every mutation (vault_ops).
+ synthetic edges: folder, temporal (±7d), tag_cooccurrence
→
4. Enrichment
Weekly algorithmic pass:
• PageRank (10 iterations)
• Louvain clustering → cluster_id
• Bridge detection
Daily fast-score: centrality shifts, access divergence, tag anomalies
→
5. Synthesis
High-signal staging: fast-score surfaces candidates for attention.
Enrichment outputs become nodes in the graph, creating new edges and further enrichment.
↑ enrichment outputs re-enter as notes → new graph edges

How Queries Use the Graph (findRelated scoring model)
BFS
Breadth-first traversal from seed node, depth 1–2. Raw score = edge weight (1.0 or 1.5 depending on edge type). Edges traversed in both directions.
÷ Hub
Hub dampening. If node degree > 50, multiply score by 50 / degree. Prevents hyper-connected nodes from dominating results. A node with 200 connections gets 0.25× weight.
⊕ Cluster
Cluster diversity. Uses Louvain cluster_id from vault_enrichment. Results spread across clusters so you don't get 25 notes from the same topic neighborhood.
× 0.1
Dormant dampening. Notes in DAMPENED_FOLDERS get 0.1× weight. Phantoms (referenced but never written) and transcripts excluded entirely. Top 25 returned.
Feedback Loops
↻ Write → Query
Every write_note immediately extracts edges and returns findRelated results. The author sees their note's neighborhood the moment they save. New edges are queryable within milliseconds.
↻ Enrichment → Query
Weekly PageRank + Louvain writes to vault_enrichment. Next findRelated call uses updated cluster_ids for diversity scoring. Queries get smarter every week without any code change.

write_note(path, content) → PUT markdown to R2 → extractEdgesFromNote() parses wikilinks, tags, frontmatter (speakers, concepts), aliases → applyOps([add_edge, upsert_node]) writes to vault_ops audit log, materializes vault_edges, upserts vault_nodes → drainDegrees() recomputes in/out_degree → returns neighborhood suggestions to caller
Extract phase: paginate R2 listing (200 notes/page, batch GET 5 at a time) → extractEdgesFromNote() per note → UPSERT vault_nodes + applyOps(edges) + INSERT/UPDATE vault_fts (FTS5).
Finalize phase: add folder edges (same-folder pairs), temporal edges (±7 day proximity), tag_cooccurrence edges (2+ shared tags) → drainDegrees() final degree computation → stale cleanup (remove old ingest_run_id rows).
24-hour cooldown · force bypasses · returns cursor for resumable pagination
syncGraph() → drain prior writers' degrees first → LIST R2 since __last_sync__ watermark → process up to SYNC_LIMIT=20 modified notes → for each: GET content, extractEdgesFromNote(), reconcileExtract(): DELETE old extract edges, INSERT desired edges, UPSERT node → update watermark → drain degrees.
1-hour cooldown · force bypasses
POST /api/ingest-triples with {s, r, o} triples → chunked 33 at a time → applyOps(add_edge, origin="ingest_triples") — vault_ops logs with origin, vault_edges materialized → drainDegrees().
Max 2000 triples per call · automation pipeline primary consumer
find_related(path, depth=2) → BFS traversal from seed node with four scoring adjustments:
• Hub dampening — degree > 50 → weight × 50/degree
• Cluster diversity — vault_enrichment.cluster_id ensures spread
• Dormant dampening — DAMPENED_FOLDERS → 0.1× weight
• Exclude phantoms & transcripts → Top 25 results by score
Daily cron (05:00 UTC) runs 4 independent detectors against D1:
• detectBridges() — edges crossing cluster boundaries
• detectAccessDivergence() — high centrality but rarely accessed
• detectCentralityShifts() — PageRank delta > 10% since last run
• detectTagAnomalies() — tags appearing in 4+ concepts with missing edges
Results written to R2 at memory/high-signal/staging/<ts>.md
CRDT: POST /api/sync-ops with ULID watermark → exchange ops bidirectionally → applyOps() incoming ops.
Weekly enrichment: Louvain clustering → PageRank (10 iterations) → bridge detection → UPDATE vault_enrichment (pagerank, cluster_id, prev_pagerank) → writeVaultSnapshot() (nodes + edges as JSONL to R2)
